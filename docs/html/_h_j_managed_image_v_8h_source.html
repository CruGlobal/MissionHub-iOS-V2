00015 /* 
00016  The managed image view is a UIView subclass that can be used just like any other view, 
00017  and it contains a UIImage object thats a managed object, and hence 
00018  can be cached, shared, and asynchronously loaded by the object manager. 
00019  So you can think of it like a UIImageView
00020  that has built in image object sharing, asynchronous loading, and caching.
00021  If you want to use HJCache for handling images (which is what its primarily designed for)
00022  you'll probably want to use this class or your own version of it.
00023  NB HJManagedImageV is an example of how to use HJCache, its not fully functional for all cases,
